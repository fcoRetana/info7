    De producto

    Historias de usuario que son para el desarrollo de GaryAdmin.

    Funcionales: comportamiento interno y externo del sistema.

    • Usabilidad
    • Confiabilidad
    • Desempeño
    • Soporte

    No Funcionales:

    Describen atributos del sistema o atributo del ambiente del sistema que no se relacionan directamente con el comportamiento interno o externo del sistema.

    De Interfaz

    un elemento externo con el que debe interactuar un sistema (restricciones de formato, tiempo u otros factores que utilice esta interacción).



    A la medida

    Historias de usuario que son para el desarrollo de sistema a la medida.

    Funcionales: comportamiento interno y externo del sistema.

    • Usabilidad
    • Confiabilidad
    • Desempeño
    • Soporte

    No Funcionales:

    Describen atributos del sistema o atributo del ambiente del sistema que no se relacionan directamente con el comportamiento interno o externo del sistema.

    De Interfaz

    un elemento externo con el que debe interactuar un sistema (restricciones de formato, tiempo u otros factores que utilice esta interacción).

    Justificación

    Sugerimos utilizar la clasificación propuesta por CMMI en el Nivel 2 - RDM 2.2 que indica: Transforme las necesidades, expectativas, restricciones e interfaces o conexiones de las partes interesadas en requisitos priorizados del cliente. También tomamos esta clasificación debido a que Somerville en su libro “Ingeniería de Software - 9ed” así lo sugiere.
